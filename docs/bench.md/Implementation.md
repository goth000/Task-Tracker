    Implementation Plan
1. Краткое описание задачи
Разработать консольное приложение-трекер задач (todo) для управления списком задач с полями id, title, due, priority, group, done. Поддержка операций CRUD (создание, чтение, обновление, удаление), фильтрация по группам, отчёт о просроченных задачах. Данные хранятся в JSON-файле data.json с ручным парсингом и валидацией.
​

2. Цель и ожидаемый результат
Цель: Создать надёжное CLI-приложение на C++ для управления задачами с хранением в JSON, обеспечивающее валидацию данных, удобный текстовый интерфейс и базовую аналитику (просроченные задачи, фильтры).

Ожидаемый результат:

Исполняемый файл todo.exe с интерактивным меню.

Пример входного/выходного файла data/data.json с задачами.

Отчёты в консоли: список всех задач, просроченные задачи, фильтр по группе.

Документация: README, бенчмарки, тесты в структуре GitHub-репозитория.
​

3. Этапы и задачи реализации
№	Этап	Задачи
1	Анализ формата данных	Изучить структуру JSON для задач [{id, title, due, priority, group, done}]. Определить валидацию: формат даты YYYY-MM-DD, приоритет {low, mid, high}, булево done. Подготовить пример data.json.
​
2	Структура данных и валидация	Создать struct task. Реализовать isValidDate, isLeapYear, проверку приоритета. Подготовить функции ввода CreateTask, RefactorTask с повтором при ошибках.
​
3	Парсинг и сохранение JSON	Написать readFile — построчный разбор JSON-массива объектов. Реализовать saveAllTasks — генерация валидного JSON с правильными запятыми и отступами. Обработать отсутствие файла.
​
4	Основные операции CRUD	Реализовать меню Start: добавление (case 1), удаление (case 2), редактирование (case 3) задач с проверками границ и сохранением после изменений.
​
5	Алгоритмы фильтрации и аналитики	Написать PrintByGroup (фильтр по group), isOverdue (сравнение дат), PrintOverdue (просроченные невыполненные задачи). Реализовать пункты меню 4–6.
​
6	UX и надёжность	Добавить checkAgree, checkDone с повторным вводом. Обработать пустые коллекции, неверные id, битые файлы. Форматировать вывод задач в табличном виде.
​
7	Производительность и бенчмарки	Замерить время фильтрации/отчёта на 10k/50k/100k задачах через std::chrono. Создать /tests/benchmark.cpp. Подготовить docs/bench.md с таблицами.
​
8	Тестирование и документация	Написать самотесты /tests/test_todo.cpp. Создать README, структуру репозитория, docs/Implementation_Plan.md. Провести финальную отладку.
​
